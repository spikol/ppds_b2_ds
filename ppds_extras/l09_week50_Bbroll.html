Review: What is OOP?

  • A method of structuring a program by bundling related properties and behaviours into individual objects
  • Components of a System (metaphor)
  • An object contains data (raw and preprocessed materials)
  • Different than Procedural Programming

Classes vs Instances

  • Classes

    • Classes are used to create user-defined data structures.

    • Classes define functions called methods, which identify the behaviours and actions that an object 

  • Instances
    • While the class is the blueprint, an instance is an object that is built from a class and contains real data

Class and .__init_() Method

  • The initial state of the object (init)
  • any number of parametersIn the body of .__init__(), there are two statements using the self variable:
  • self.name = name creates an attribute called name and assigns to it the value of the name parameter.
  • self.age = age creates an attribute called age and assigns to it the value of the age parameter.
class Dog:
    """Class attributes"""
    species = "Canis familiaris"

    def __init__(self, name, age):
        self.name = name
        self.age = age

Passing Arguments

  • To pass arguments to the name and age parameters, put values into the parentheses after the class name: 
class Dog:
    """Class attributes"""
    species = "Canis familiaris"

    def __init__(self, name, age):
        self.name = name
        self.age = age

buddy = Dog("buddy", 9)
mikey = Dog("mikey", 3)

Instance Methods

class Dog:
    species = "Canis familiaris"

    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Instance method
    def description(self):
        return "{} is {} years old".format(self.name, self.age)

    # Another instance method
    def speak(self, sound):
        return "{} says {}". format(self.name, sound)

buddy = Dog("buddy", 9)
mikey = Dog("mikey", 3)
bob = Dog("bob",2)

print(buddy)
print(bob.description())
print(mikey.speak("woof woof"))
